## What is SecureCommands?

SecureCommands is a powerful Minecraft plugin designed to protect your server's most critical commands from unauthorized access. Whether you're running a Spigot or Paper server, SecureCommands adds an essential security layer by requiring authentication before executing protected commands.

<Note>
  SecureCommands v2.3 is compatible with Minecraft 1.18.2+ and requires Java 17 or higher.
</Note>

## Why SecureCommands?

Server security is crucial, especially when dealing with powerful administrative commands. SecureCommands solves common security challenges:

- **Prevent Unauthorized Access**: Block dangerous commands like `/op`, `/ban`, and `/kick` until users authenticate
- **Whitelist System**: Control exactly who can access protected commands
- **Session-Based Authentication**: Users authenticate once per session using `/sclogin`
- **Premium Account Detection**: Automatically authenticate premium (paid) Minecraft accounts
- **Discord Integration**: Receive real-time security alerts when someone tries to use protected commands
- **OP Status Protection**: Automatically remove OP status from non-whitelisted players

## How It Works

SecureCommands operates on a simple but effective security model:

<Steps>
  <Step title="Configure Protected Commands">
    Define which commands require authentication in your `config.yml` file.
  </Step>
  
  <Step title="Whitelist Trusted Players">
    Add players to the whitelist using `/sc add <player>`.
  </Step>
  
  <Step title="Players Set Secure Codes">
    Whitelisted players create their own secure codes with `/sc setcode <password>`.
  </Step>
  
  <Step title="Authentication Required">
    When a player tries to use a protected command, they must first authenticate using `/sclogin <password>`.
  </Step>
  
  <Step title="Session-Based Access">
    Once authenticated, players can use protected commands until they disconnect from the server.
  </Step>
</Steps>

<Warning>
  Protected commands cannot be executed until the player authenticates each session. This ensures maximum security even if someone gains temporary access to a player's account.
</Warning>
