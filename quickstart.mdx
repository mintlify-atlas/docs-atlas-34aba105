## Prerequisites

Before you begin, ensure you have:

- Spigot or Paper server running version 1.18.2 or higher
- Java 17 or higher installed
- MySQL database server
- Server admin/OP access

<Note>
If you haven't installed SecureCommands yet, check out the [Installation Guide](/installation) first.
</Note>

## Step 1: Configure Your First Protected Command

Let's start by protecting the `/op` command to prevent unauthorized operator access.

<Steps>
  <Step title="Edit config.yml">
    Open `plugins/SecureCommands/config.yml` and configure the blocked commands:
    
    ```yaml config.yml
    # Commands to protect
    blocked-commands:
      - "/op"
      - "/deop"
      - "/ban"
      - "/pardon"
    ```
  </Step>
  
  <Step title="Add yourself to the whitelist">
    Add your username to the whitelist so you can set up authentication:
    
    ```yaml config.yml
    whitelist:
      - "YourUsername"
    ```
  </Step>
  
  <Step title="Reload the plugin">
    Apply the configuration changes:
    
    ```bash
    /sc reload
    ```
  </Step>
</Steps>

## Step 2: Set Up Your Secure Code

Now that you're whitelisted, create your authentication password.

```bash
/sc setcode MySecurePassword123
```

**Response**:
```
[SecureCommands] Secure code set successfully.
```

<Warning>
Choose a strong password with at least 8 characters, mixing letters, numbers, and symbols. This password is hashed using BCrypt and cannot be recovered if lost.
</Warning>

## Step 3: Authenticate

Now test the authentication system by trying to execute a protected command:

```bash
# Try to execute a blocked command without authentication
/op SomePlayer
```

**Response**:
```
[SecureCommands] This command is protected. Please use /sclogin <password>.
```

**Authenticate with your secure code**:

```bash
/sclogin MySecurePassword123
```

**Response**:
```
[SecureCommands] You are now authenticated!
```

Now you can execute protected commands until you disconnect or the server restarts.

## Step 4: Add Other Players

Add trusted staff members to the whitelist and have them set up their own secure codes.

### Using In-Game Commands

```bash
# Add a player to the whitelist
/sc add StaffMember1

# The player can now set their own secure code
# (StaffMember1 runs this)
/sc setcode TheirSecurePassword456
```

### Using Configuration File

Edit `config.yml`:

```yaml config.yml
whitelist:
  - "YourUsername"
  - "StaffMember1"
  - "StaffMember2"
```

Then reload:

```bash
/sc reload
```

## Common Workflows

### Authenticating After Join

Every time you join the server, you'll need to authenticate before using protected commands:

<Steps>
  <Step title="Join the server">
    Connect to your Minecraft server
  </Step>
  
  <Step title="Authenticate">
    Run the login command:
    
    ```bash
    /sclogin YourPassword
    ```
  </Step>
  
  <Step title="Execute protected commands">
    You can now use `/op`, `/ban`, and other blocked commands until you disconnect
  </Step>
</Steps>

### Removing a Player's Access

If you need to revoke someone's access:

```bash
# Remove from whitelist
/sc remove StaffMember1

# Reset their secure code
/sc resetcode StaffMember1
```

<Info>
Removing a player from the whitelist also revokes their OP status automatically if they have it.
</Info>

### Blocking Additional Commands

Add more commands to the protection list:

```bash
# Add a command to the blocked list
/sc addcommand /gamemode

# Remove a command from the blocked list
/sc removecommand /gamemode
```

Or edit `config.yml` directly:

```yaml config.yml
blocked-commands:
  - "/op"
  - "/deop"
  - "/ban"
  - "/pardon"
  - "/gamemode"
  - "/give"
  - "/tp"
```

## Premium Player Auto-Authentication

If you're running an **offline-mode server**, premium Minecraft accounts can be automatically authenticated.

### How It Works

<Steps>
  <Step title="Premium player joins">
    When a whitelisted player with a legitimate Minecraft account joins, SecureCommands checks their premium status via the Mojang API
  </Step>
  
  <Step title="Automatic authentication">
    If verified as premium, they're automatically authenticated without needing `/sclogin`
  </Step>
  
  <Step title="Execute commands">
    Premium players can immediately use protected commands
  </Step>
</Steps>

### Disabling Premium Check

If you want a premium player to use password authentication instead:

```bash
/sc premium PlayerName
```

This toggles the premium check on/off for that player.

<Note>
Learn more about premium player features in the [Premium Players](/features/premium-players) guide.
</Note>

## Discord Notifications

Set up Discord alerts to monitor security events in real-time.

### Quick Setup

<Steps>
  <Step title="Create a Discord webhook">
    In your Discord server, go to a channel → **Edit Channel** → **Integrations** → **Webhooks** → **New Webhook**
  </Step>
  
  <Step title="Copy the webhook URL">
    Copy the webhook URL from Discord
  </Step>
  
  <Step title="Add to config.yml">
    ```yaml config.yml
    discord:
      enabled: true
      security-webhook-url: "YOUR_WEBHOOK_URL_HERE"
    ```
  </Step>
  
  <Step title="Reload the plugin">
    ```bash
    /sc reload
    ```
  </Step>
</Steps>

Now you'll receive Discord alerts for:
- Failed login attempts
- Blocked command executions
- Plugin updates

## Testing Your Setup

Verify everything is working correctly:

<Steps>
  <Step title="Test command blocking">
    Without authenticating, try a blocked command:
    
    ```bash
    /op TestPlayer
    ```
    
    **Expected**: Command blocked, message shown
  </Step>
  
  <Step title="Test authentication">
    Authenticate with your secure code:
    
    ```bash
    /sclogin YourPassword
    ```
    
    **Expected**: Success message
  </Step>
  
  <Step title="Test command execution">
    Execute the blocked command again:
    
    ```bash
    /op TestPlayer
    ```
    
    **Expected**: Command executes successfully
  </Step>
  
  <Step title="Test failed login">
    Reconnect and try an incorrect password:
    
    ```bash
    /sclogin WrongPassword
    ```
    
    **Expected**: Error message + Discord alert (if configured)
  </Step>
</Steps>

## Troubleshooting

<Accordion title="Commands are still blocked after authentication">
  Make sure you received the success message after `/sclogin`. If not:
  
  - Check that you're whitelisted in `config.yml`
  - Verify your secure code is set with `/sc setcode`
  - Ensure the password is correct (case-sensitive)
  - Check server logs for authentication errors
</Accordion>

<Accordion title="Can't set a secure code">
  Common issues:
  
  - **Not whitelisted**: Add yourself to the whitelist in `config.yml`
  - **Already have a code**: Use `/sc resetcode YourName` (console only) to reset
  - **Premium player**: Premium players don't need secure codes for auto-authentication
</Accordion>

<Accordion title="Database connection failed">
  If you see database errors on startup:
  
  1. Verify MySQL is running
  2. Check database credentials in `config.yml`
  3. Ensure the database exists: `CREATE DATABASE secure_commands;`
  4. Verify user has permissions: `GRANT ALL PRIVILEGES ON secure_commands.* TO 'user'@'localhost';`
  
  See the [Database Configuration](/configuration/database) guide for detailed setup.
</Accordion>
