## Overview

SecureCommands provides a public API for developers who want to integrate with the plugin or extend its functionality. This guide covers all available public methods and how to use them.

## Getting Started

### Add SecureCommands as a Dependency

**Maven:**
```xml
<dependency>
    <groupId>org.pablito</groupId>
    <artifactId>SecureCommands</artifactId>
    <version>2.3</version>
    <scope>provided</scope>
</dependency>
```

**Gradle:**
```gradle
compileOnly 'org.pablito:SecureCommands:2.3'
```

### Declare Dependency in plugin.yml

```yaml
name: YourPlugin
version: 1.0
main: com.example.YourPlugin
depend: [SecureCommands]
# or use softdepend if optional
softdepend: [SecureCommands]
```

## Core API Methods

### Getting the Plugin Instance

The main entry point for accessing SecureCommands functionality.

```java
import org.pablito.secureCommands.SecureCommands;

public class YourPlugin extends JavaPlugin {
    private SecureCommands secureCommands;
    
    @Override
    public void onEnable() {
        // Get the SecureCommands instance
        secureCommands = SecureCommands.getInstance();
        
        if (secureCommands == null) {
            getLogger().warning("SecureCommands not found!");
            return;
        }
        
        getLogger().info("Successfully hooked into SecureCommands!");
    }
}
```

**Method Signature:**
```java
public static SecureCommands getInstance()
```

**Returns:** The active `SecureCommands` instance, or `null` if the plugin is not loaded.

**Location:** `SecureCommands.java:133-135`

### DatabaseManager

Access database operations for player authentication and secure codes.

#### Get DatabaseManager Instance


**Returns:** The `DatabaseManager` instance for database operations.

**Location:** `SecureCommands.java:137-139`

#### Check if Secure Code is Set

Verify if a player has already set their secure code.

```java
boolean hasCode = databaseManager.isSecureCodeSet("Steve");
if (hasCode) {
    player.sendMessage("You already have a secure code set!");
}
```

**Method Signature:**
```java
public boolean isSecureCodeSet(String player)
```

**Parameters:**
- `player` - The player's name (case-insensitive)

**Returns:** `true` if the player has a secure code set, `false` otherwise.

**Location:** `DatabaseManager.java:54-63`

#### Set Secure Code

Set a new secure code for a player.

```java
boolean success = databaseManager.setSecureCode("Steve", "mySecurePassword123", false);
if (success) {
    player.sendMessage("Secure code set successfully!");
} else {
    player.sendMessage("You already have a secure code. Reset it first.");
}
```

**Method Signature:**
```java
public boolean setSecureCode(String player, String code, boolean isPremium)
```

**Parameters:**
- `player` - The player's name
- `code` - The password to set (will be hashed with BCrypt)
- `isPremium` - Whether the account is premium (true) or cracked (false)

**Returns:** `true` if successful, `false` if the player already has a code set.

**Location:** `DatabaseManager.java:65-83`

<Warning>
Passwords are automatically hashed with BCrypt (cost factor 12). Never store or log plaintext passwords.
</Warning>

## Example Integration

### Complete Example: Custom Authentication Command

Here's a complete example of integrating with SecureCommands to create a custom authentication system:

```java
package com.example.customplugin;

import org.bukkit.command.Command;
import org.bukkit.command.CommandExecutor;
import org.bukkit.command.CommandSender;
import org.bukkit.entity.Player;
import org.bukkit.plugin.java.JavaPlugin;
import org.pablito.secureCommands.SecureCommands;
import org.pablito.secureCommands.managers.DatabaseManager;
import org.pablito.secureCommands.listeners.CommandBlocker;

public class CustomAuthPlugin extends JavaPlugin implements CommandExecutor {
    
    private SecureCommands secureCommands;
    private DatabaseManager databaseManager;
    private CommandBlocker commandBlocker;
    
    @Override
    public void onEnable() {
        // Get SecureCommands instance
        secureCommands = SecureCommands.getInstance();
        
        if (secureCommands == null) {
            getLogger().severe("SecureCommands not found! Disabling plugin.");
            getServer().getPluginManager().disablePlugin(this);
            return;
        }
        
        // Get manager instances
        databaseManager = secureCommands.getDatabaseManager();
        commandBlocker = secureCommands.getCommandBlocker();
        
        // Register custom command
        getCommand("customauth").setExecutor(this);
        
        getLogger().info("Successfully hooked into SecureCommands!");
    }
    
    @Override
    public boolean onCommand(CommandSender sender, Command command, 
                            String label, String[] args) {
        if (!(sender instanceof Player)) {
            sender.sendMessage("Only players can use this command.");
            return true;
        }
        
        Player player = (Player) sender;
        
        if (args.length < 1) {
            player.sendMessage("Usage: /customauth <password>");
            return true;
        }
        
        String enteredPassword = args[0];
        String playerName = player.getName();
        
        // Check if player has a secure code set
        if (!databaseManager.isSecureCodeSet(playerName)) {
            player.sendMessage("You don't have a secure code set. Contact an admin.");
            return true;
        }
        
        // Verify the password
        if (databaseManager.isSecureCodeCorrect(playerName, enteredPassword)) {
            // Authenticate the player
            commandBlocker.authenticatePlayer(player.getUniqueId());
            player.sendMessage("Successfully authenticated!");
            
            // Custom logic here
            grantCustomPermissions(player);
        } else {
            player.sendMessage("Incorrect password!");
        }
        
        return true;
    }
    
    private void grantCustomPermissions(Player player) {
        // Your custom logic after successful authentication
        player.sendMessage("Custom permissions granted!");
    }
}
```

### Example: Programmatically Managing Secure Codes

```java
import org.pablito.secureCommands.SecureCommands;
import org.pablito.secureCommands.managers.DatabaseManager;

public void setupNewAdmin(String playerName, String temporaryPassword) {
    DatabaseManager db = SecureCommands.getInstance().getDatabaseManager();
    
    // Remove any existing code
    if (db.isSecureCodeSet(playerName)) {
        db.resetSecureCode(playerName);
    }
    
    // Set new temporary password
    boolean success = db.setSecureCode(playerName, temporaryPassword, false);
    
    if (success) {
        getLogger().info("Set up secure code for " + playerName);
        // Send password via email or Discord
        notifyAdmin(playerName, temporaryPassword);
    }
}
```

## Event Listening

While SecureCommands doesn't provide custom events (yet), you can listen to Bukkit events to integrate with authentication:

```java
import org.bukkit.event.EventHandler;
import org.bukkit.event.Listener;
import org.bukkit.event.player.PlayerJoinEvent;
import org.pablito.secureCommands.SecureCommands;

public class AuthListener implements Listener {
    
    @EventHandler
    public void onPlayerJoin(PlayerJoinEvent event) {
        Player player = event.getPlayer();
        
        // Check if player needs to authenticate
        Set<UUID> pending = SecureCommands.getInstance().getPendingAuthentication();
        
        if (pending.contains(player.getUniqueId())) {
            player.sendMessage("Please authenticate with /sclogin <password>");
            
            // Remove from spawn or apply restrictions
            applyAuthenticationRestrictions(player);
        }
    }
    
    private void applyAuthenticationRestrictions(Player player) {
        // Your custom logic (e.g., teleport to auth area, freeze player)
    }
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Always Check for Null">
    Always verify that `SecureCommands.getInstance()` returns a non-null value before using the API:
    
    ```java
    SecureCommands sc = SecureCommands.getInstance();
    if (sc == null) {
        // Handle SecureCommands not being loaded
        return;
    }
    ```
  </Accordion>
  
  <Accordion title="Use softdepend for Optional Integration">
    If SecureCommands integration is optional, use `softdepend` in your plugin.yml and check for its presence:
    
    ```yaml
    softdepend: [SecureCommands]
    ```
    
    ```java
    if (getServer().getPluginManager().getPlugin("SecureCommands") != null) {
        // SecureCommands is available
        setupSecureCommandsIntegration();
    }
    ```
  </Accordion>
  
  <Accordion title="Never Store Plaintext Passwords">
    Always use `DatabaseManager.setSecureCode()` to store passwords. Never store plaintext passwords in your own database or log them.
  </Accordion>
  
  <Accordion title="Respect Plugin Configuration">
    Check the player's whitelist status and authentication state before granting access to privileged operations.
  </Accordion>
</AccordionGroup>
